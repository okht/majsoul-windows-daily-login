# 雀魂 Windows 每日静默打开设计

- 日期：2026-07-16
- 修订：2026-07-19（本地时区调度；Task 8 隐私扫描与验收回执）
- 状态：实现完成、本机验收待用户完成；调度语义以本机系统时区为准
- 目标页面：`https://game.maj-soul.com/1/`
- 目标平台：用户家中的 Windows 电脑

## 1. 目标

每天使用家庭网络和本机 Microsoft Edge 的独立浏览器会话打开雀魂页面，复用用户预先建立的登录状态，等待页面自动进入大厅并触发每日奖励。

系统必须满足以下约束：

1. 每天按**本机 Windows 系统时区**的 10:00–12:30 随机运行一次（当地上午，不强制北京时间）。
2. 不主动唤醒睡眠中的电脑。
3. 如果当天随机任务被错过，本地 12:30 后在用户登录或解锁 Windows 且网络可用时补跑。
4. 运行过程不显示窗口、不抢焦点，进程使用后台优先级。
5. 定时运行期间不执行任何鼠标点击、键盘输入、表单填写或合成输入事件。
6. 页面需要人工操作时立即停止，当天进入人工阻塞状态。
7. 失败或需要人工处理时发送纯文字 Gmail；成功时不发送邮件。
8. 不保存雀魂邮箱、雀魂密码或页面截图。
9. 不因系统时区不是「中国标准时间」而拒绝安装或注册。

## 2. 非目标

- 不支持 iOS。
- 不使用 AWS、GitHub Actions 或其他云端运行环境。
- 不执行对局、任务领取、活动操作或任何游戏内交互。
- 不处理验证码，不绕过访问限制，不加入指纹伪装、代理或反检测功能。
- 不承诺平台无法识别自动访问。
- 不尝试恢复前一天已经错过的奖励。

## 3. 方案选择

### 采用方案：独立 Edge 会话＋Windows 任务计划

使用系统安装的 Microsoft Edge 和专用用户数据目录。首次设置时由用户在可见窗口中手动登录雀魂；日常任务使用同一个专用目录静默打开页面。

采用该方案的原因：

- 家庭网络和本机浏览器环境与用户日常环境一致。
- 独立浏览器目录不会锁定或污染用户日常 Edge 配置。
- Windows 任务计划原生支持每日触发、随机延迟、错过后补跑和联网条件。
- 雀魂凭证不进入程序、配置文件、日志或 Windows 任务定义。

### 未采用方案

1. 当前桌面上最小化普通浏览器：可能抢焦点、显示任务栏图标并影响用户操作。
2. 独立 Windows 用户会话：隔离更强，配置和运行复杂度更高，仅在静默 Edge 兼容性验证失败后重新评估。
3. 云端浏览器：云 IP、浏览器环境和服务条款风险更高，已从范围移除。

## 4. 用户流程

### 4.1 一次性设置

1. 安装程序创建 `%LOCALAPPDATA%\MajSoulDaily`。
2. 设置程序用专用 Edge 目录打开雀魂页面。
3. 用户亲自完成登录并确认已经进入大厅。
4. 用户关闭设置窗口。
5. 设置程序验证专用会话可以再次自动进入页面，并注册 Windows 计划任务。
6. 用户提供 Gmail 地址和 Gmail 应用专用密码；应用专用密码写入 Windows 凭据管理器。

设置程序只负责打开专用浏览器。登录、确认及任何按钮操作均由用户完成。

### 4.2 会话修复

收到人工操作通知后，用户运行会话修复入口。修复入口以可见方式打开专用 Edge 会话，用户手动完成所需操作并确认大厅已经出现。修复入口随后清除当天的 `BLOCKED_MANUAL` 状态并执行一次无输入验证。

### 4.3 每日运行

1. 任务计划触发后台运行器。
2. 运行器检查 Windows 会话是否已经解锁，再检查当天状态和互斥锁。
3. 随机任务到期但会话仍锁定时记录 `PENDING_DUE`，不启动浏览器。
4. 当天已成功或已进入人工阻塞状态时立即退出。
5. 运行器确认当前允许执行，并确认目标站点可连接。
6. 运行器使用专用 Edge 会话打开目标页面。
7. 运行器只被动观察页面状态，最长等待 3 分钟。
8. 确认自动进入大厅后记录成功，关闭浏览器并退出。
9. 如果页面要求点击、登录、确认或验证，立即关闭浏览器，记录人工阻塞并发送 Gmail。

## 5. 调度规则

系统注册两个计划任务（主任务与补跑任务），均调用同一安装目录中的无窗口启动器。  
所有时刻均指**本机系统时区的墙上时钟**（与 Windows「日期和时间」时区一致）。电脑设在中国即为中国当地上午；设在其他时区即为该时区当地上午，**不会**把其他时区硬转换成北京时间。

### 5.1 主触发器

- 每天本地 10:00 触发。
- `RandomDelay` 设置为 2 小时 30 分钟。
- 实际开始时间落在本地 10:00–12:30。

### 5.2 补跑触发器

- 启用 `StartWhenAvailable`，允许错过计划时间后补跑。
- 启用 `RunOnlyIfNetworkAvailable`。
- 用户登录 Windows 或工作站解锁时触发一次检查。
- 每天本地 12:30 开始，每 15 分钟执行一次轻量检查，持续到本地 23:45。
- 本地 12:30 前由登录或联网事件触发的补跑检查直接退出，不提前破坏随机窗口。
- 随机任务已经到期且状态为 `PENDING_DUE` 时，本地 12:30 前解锁也可以立即补跑。
- 本地 12:30 后尚未成功时，登录或解锁 Windows 且联网后立即运行；联网事件未被系统捕获时，15 分钟轮询负责兜底。

### 5.3 运行约束

- `WakeToRun` 关闭，不唤醒电脑。
- 多实例策略使用 `IgnoreNew`。
- 任务优先级使用后台级别 8。
- 单次执行上限 10 分钟。
- Windows 会话处于锁定状态时不启动浏览器。
- 当天成功后，后续补跑只读取状态并立即退出。
- 午夜后只处理新的**本机本地**自然日（与系统时区一致）。
- 安装/注册**不要求** `China Standard Time`；用户可在任意 Windows 时区下使用本机当地窗口。

## 6. 组件与边界

### 6.1 安装器

职责：检查 Node.js 与 Edge、创建目录、注册或更新计划任务、验证任务设置。

不负责：登录雀魂、保存雀魂凭证、执行页面交互。

### 6.2 会话设置器

职责：以可见方式打开专用 Edge 会话，让用户手动建立登录状态。

不负责：自动填写、点击或验证账号。

### 6.3 调度门卫

职责：根据本机本地时刻、触发来源、当天状态、联网状态和互斥锁决定是否运行浏览器。

输出：`RUN`、`SKIP_SUCCESS`、`SKIP_BLOCKED`、`SKIP_BEFORE_WINDOW` 或 `SKIP_OFFLINE`。

### 6.4 浏览器运行器

职责：使用 `playwright-core` 调用系统 Edge，打开目标 URL，等待并被动观察。

允许的浏览器动作只有：启动、导航、读取状态、关闭。

禁止的动作包括：`click`、`dblclick`、`tap`、`press`、`type`、`fill`、`dispatchEvent` 及任何坐标输入。

### 6.5 大厅检测器

检测器采用只读信号：

1. 优先读取可访问的 URL、标题、DOM 或辅助功能标记。
2. 页面主要由 Canvas 渲染时，只在内存中读取当前帧并计算不可逆的视觉特征；不把帧写入磁盘或邮件。
3. 一次性设置期间，在用户确认大厅后保存紧凑的视觉特征数据，不保存原始图片。
4. 180 秒内匹配到大厅特征才返回 `SUCCESS`。
5. 发现登录、确认、验证码或其他人工操作界面时返回 `MANUAL_ACTION_REQUIRED`。
6. 无法确认大厅时返回 `LOBBY_TIMEOUT`，不进行试探性点击。

### 6.6 状态存储

状态保存在 `%LOCALAPPDATA%\MajSoulDaily\state`，按**本机本地日期**（`YYYY-MM-DD`）分文件，使用原子替换写入。

状态值：

- `PENDING_DUE`
- `RUNNING`
- `SUCCESS`
- `FAILED_TRANSIENT`
- `BLOCKED_MANUAL`

状态只包含日期、时间、结果、错误分类、通知去重标记和运行版本，不包含账号信息或页面内容。

### 6.7 Gmail 通知器

职责：通过 Gmail SMTP 发送纯文字失败通知。

凭据：Gmail 应用专用密码存入 Windows 凭据管理器，配置文件只保存发件地址和收件地址。

邮件内容：

- 发生时间
- 设备名称
- 失败类型
- 当前阶段
- 已执行次数
- 建议操作
- 本地日志路径

通知策略：

- `MANUAL_ACTION_REQUIRED` 或登录状态失效时立即通知，保持阻塞直到用户完成会话修复。
- 网络或浏览器瞬时失败允许后续补跑；相同错误每天最多通知一次。
- 成功不通知。
- 邮件不附带截图、会话信息或账号凭证。

## 7. 错误处理

| 情况 | 处理 |
| --- | --- |
| 当天已经成功 | 立即退出 |
| 任务重复触发 | 互斥锁保留现有实例，新实例退出 |
| 目标站点暂时离线 | 记录瞬时失败，等待下一个补跑 |
| Edge 启动失败 | 重试一次；仍失败则通知 |
| 登录状态失效 | 记录 `BLOCKED_MANUAL`，通知并等待用户运行会话修复 |
| 页面需要点击或验证 | 记录 `BLOCKED_MANUAL`，通知并等待用户运行会话修复 |
| 180 秒内无法确认大厅 | 记录失败并通知，不点击 |
| Gmail 发送失败 | 写入本地日志；后续触发最多补发一次 |
| 静默 Edge 无法加载 Canvas/WebGL | 停止上线，回到设计评审，不自动切换可见窗口 |

## 8. 隐私与安全

- 雀魂邮箱和密码始终由用户在官方页面中手动输入。
- 专用浏览器目录仅供本项目使用，不复用用户日常 Edge 配置。
- Gmail 应用专用密码通过 Windows 凭据管理器保护。
- 日志统一脱敏，不记录 Cookie、Local Storage、请求正文、邮箱密码或页面截图。
- 日志保留 14 天，随后自动删除。
- 卸载时删除任务、程序文件、状态和 Gmail 凭据；专用 Edge 会话由用户选择是否删除。
- 自动访问仍可能受雀魂或 Yostar 服务条款限制。程序不包含降低可检测性或绕过平台控制的机制。

## 9. 测试与验收

### 9.1 自动测试

1. 调度门卫覆盖本地 10:00 前、随机窗口、锁屏到期、解锁补跑、本地 12:30 后、本地午夜切换、成功跳过和人工阻塞。
2. 状态存储覆盖原子写入、损坏文件恢复和多进程竞争。
3. 通知器覆盖纯文字内容、每日去重和凭据缺失。
4. 任务定义测试确认 `RandomDelay=PT2H30M`、`StartWhenAvailable=true`、联网条件、`WakeToRun=false`、优先级 8 和 `IgnoreNew`。
5. 静态安全测试扫描运行代码，禁止所有点击、键盘和合成输入 API。
6. 浏览器集成测试使用本地模拟页面覆盖自动大厅、人工按钮、登录过期、超时和崩溃。

### 9.2 本机验收

1. 用户在专用 Edge 会话中完成一次手动登录。
2. 手动运行计划任务，确认无窗口、无焦点切换、无鼠标键盘输入。
3. 确认进入大厅后状态写为 `SUCCESS`，再次运行立即退出。
4. 使用模拟登录页验证任务停止且 Gmail 收到纯文字通知。
5. 关闭网络后触发任务，再恢复网络，确认当天补跑。
6. 随机任务到期时保持锁屏，确认浏览器没有启动且状态为 `PENDING_DUE`。
7. 解锁并联网，确认任务立即补跑。
8. 错过 12:30 后登录 Windows，确认联网后补跑。
9. 确认任务不会主动唤醒睡眠中的电脑。

### 9.3 上线门槛

只有同时满足以下条件才注册每日正式任务：

- 系统 Edge 能在静默模式下加载目标页面和 Canvas/WebGL。
- 专用会话能够自动进入大厅。
- 大厅检测器连续 3 次正确判断成功。
- 人工操作页面连续 3 次均停止，期间没有输入事件。
- Gmail 失败通知已实际到达。
- 所有自动测试通过。

任一条件失败时保持未上线状态，并向用户说明具体阻碍。
